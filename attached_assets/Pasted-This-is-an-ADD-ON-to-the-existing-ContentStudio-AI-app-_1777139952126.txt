FEATURE 3 — Update /api/generate-video-prompts Input
The API now receives the voiceover and BGM data that was selected in the inline panel:
{
  "story": "object",
  "style": "string — from story settings",
  "duration": 15,
  "part": 1,
  "totalParts": 4,
  "previousLastFrame": "null or string",
  "voiceoverLanguage": "hinglish or null",
  "voiceoverTone": "cinematic or null",
  "bgmStyle": "Cinematic Orchestral or null",
  "bgmTempo": "110 BPM or null",
  "bgmInstruments": ["strings", "percussion", "piano"]
}
If voiceoverLanguage is not "none" and no pre-written script exists for this part, Claude auto-writes the voiceover script for that part based on the story act that maps to this part number.
Auto-voiceover logic in system prompt:
If voiceoverScript is not provided but voiceoverLanguage is set:
- Look at which story act maps to this part number
- Write a voiceover script that captures the emotional core of that act
- Word count: duration_seconds × 2.5 words (for cinematic/slow tone) or × 3.2 (for energetic tone)
- Language: follow the voiceoverLanguage setting exactly
- Embed this auto-written script into the [VOICEOVER: ...] block of the copyablePrompt
